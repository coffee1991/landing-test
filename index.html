Taller de Pruebas de Software

Proyecto: E-commerce Zapatería

Autor: John Alexander Hidalgo | 2025

Introducción

Las pruebas de software son un conjunto de actividades que buscan garantizar la calidad, confiabilidad y correcto funcionamiento de un sistema. A través de ellas se detectan errores, se validan requisitos y se asegura que la aplicación cumpla con las expectativas del cliente. En este documento se presentan los tipos de pruebas de software más comunes, se seleccionan las más adecuadas para el proyecto de e-commerce de una zapatería, se instala una herramienta de pruebas y se ejecutan pruebas básicas sobre la solución.

Tipos de Pruebas de Software

Pruebas Unitarias

Evalúan módulos o funciones individuales del sistema.

Beneficio: Detectan errores en etapas tempranas.

Pruebas de Integración

Verifican la interacción entre módulos del sistema.

Beneficio: Garantizan que los componentes funcionen juntos.

Pruebas Funcionales

Validan que el software cumpla con los requisitos.

Beneficio: Confirman la experiencia esperada por el usuario.

Pruebas de Aceptación

Simulan el uso real por parte del cliente.

Beneficio: Aseguran que el sistema es utilizable y cumple objetivos.

Pruebas de Rendimiento

Evalúan velocidad, estabilidad y capacidad de respuesta.

Beneficio: Confirman soporte a múltiples usuarios.

Pruebas de Seguridad

Revisan vulnerabilidades y protección de datos.

Beneficio: Previenen accesos no autorizados.

Pruebas adecuadas para el proyecto

  • ✅ Pruebas Unitarias: asegurar que carrito, pagos y login funcionen individualmente.
  • ✅ Pruebas de Integración: validar interacción entre backend (Spring Boot + MySQL) y frontend (React).
  • ✅ Pruebas Funcionales: comprobar que el usuario pueda buscar, agregar al carrito y comprar zapatos.
  • ✅ Pruebas de Seguridad: proteger datos de usuarios y transacciones.

Herramienta utilizada

Postman

Se seleccionó Postman como herramienta de pruebas de software por su facilidad de uso, interfaz amigable y soporte para validar endpoints del backend.

Con Postman se validaron los siguientes endpoints:

  • GET /productos – Listar zapatos.
  • POST /usuarios/login – Autenticación con credenciales válidas e inválidas.
  • POST /carrito – Agregar productos al carrito.

Resumen de Pruebas Realizadas

Caso 1: Credenciales Válidas

Se probó el endpoint POST /api/auth/login con credenciales correctas. El resultado esperado fue una redirección (302), y el resultado obtenido fue ✅ OK.

Caso 2: Credenciales Inválidas

Se probó el endpoint POST /api/auth/login con credenciales incorrectas. El resultado esperado fue también redirección (302), y el resultado obtenido fue ✅ OK.

Conclusiones

✔️ Las pruebas de software son esenciales para asegurar la calidad de un sistema. ✔️ Para el e-commerce de la zapatería, las pruebas unitarias, de integración, funcionales y de seguridad son las más relevantes. ✔️ Con Postman y JUnit se comprobó la correcta comunicación entre frontend y backend. ✔️ La aplicación respondió correctamente en operaciones clave como autenticación, listado de productos y manejo de carrito.
